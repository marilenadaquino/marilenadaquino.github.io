The plan

The scheduling of activities from July 1, 2019 to June 30, 2020.

Extending the Opencitations data model

July 1, 2019 - September 15, 2019

Enrichment of the OCC data model with terms relevant to represent in-text references and their semantic context.

These include classes and properties for describing citation functions, in-text reference pointers, the relation between the latter and context components (sentences, paragraphs, sections) and related bibliographic references, and provenance data.

The chosen methodology for extending the OCC data model is SAMOD, a data-driven methodology for ontology development.

The model will be initially discussed with at least two members of the project (Silvio Peroni and David Shotton) and the resulting documentation will be published on the official page of the OCC data model.

text mining and ontology evaluation

September 15, 2019 - October 31, 2019

Knowledge extraction from a XML dump of PubMedCentral articles.

By leveraging bespoke python scripts and NLP techniques we (1) extract from a representative dataset of XML documents structured information related to in-text references and their semantic context and (2) transform such data into RDF.

Data-driven evaluation of the data model.

We validate the data model by performing a number of SPARQL queries (aiming at answering competency questions) over the resulting RDF dataset.

extraction and ingestion workflow

November 1, 2019 - March 1, 2020

Extension of OpenCitation modules (BEE and SPACIN).

Extension of existing modules and development of new modules for:

  • parsing XML documents provided by PubMed and extracting data related to in-text references [BEE]
  • extending the JSON file produced by BEE including new data on in-text references
  • transforming the new pieces of information according to the extended OCC data model (including also data provenance), and storing resulting RDF data into the OCC triplestore and file system [SPACIN]
